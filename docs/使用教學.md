## 1. 登入平台

**步驟 1.1 — 進入登入頁面**

開啟瀏覽器，前往 https://business-console.zeabur.app/login

```
[截圖 1.1: 登入頁面，顯示「business-console」標題、
密碼輸入欄位及「登入」按鈕]
```

**步驟 1.2 — 輸入密碼**

在密碼欄位輸入管理員密碼，然後點擊「**登入**」按鈕。

> **提示：** 如忘記密碼，請聯絡系統管理員重設。

**步驟 1.3 — 主畫面**

登入成功後，將進入主畫面（外撥活動列表）。

```
[截圖 1.3: 主畫面，左側顯示「外撥」和「入線」導航選單，
右側顯示活動列表及頂部的行業範本選擇區]
```

**介面說明：**

| 區域 | 說明 |
|------|------|
| 左側導航 | 外撥活動、入線熱線、設定、登出 |
| 頂部範本列 | 快速選擇行業（餐廳、美容院、保險等）|
| 活動列表 | 顯示進行中、已完成、草稿的活動 |
| 右上角「新增」| 建立新活動或新熱線 |

## 2. 外撥通話活動

外撥通話讓 AI 自動致電您的客戶，進行訂座確認、預約提醒等工作。

### 2.1 建立活動範本

範本儲存常用的 AI 語音及腳本設定，下次建立活動時直接選用，無需重新設定。

**步驟 2.1.1 — 進入範本管理**

點擊頂部導航的「**管理範本**」連結（在新增活動頁面的第一步），或透過主畫面進入。

```
[截圖 2.1.1: 管理範本頁面，顯示系統內建範本列表
（餐廳、美容院、保險、旅行社、醫療診所、地產）
以及「+ 新增範本」按鈕]
```

**步驟 2.1.2 — 新增自訂範本**

點擊「**+ 新增範本**」按鈕，填寫以下資料：

```
[截圖 2.1.2: 新增範本表單，顯示各輸入欄位]
```

| 欄位 | 說明 | 範例 |
|------|------|------|
| **Emoji** | 範本圖示 | 🍽️ |
| **範本名稱** * | 識別用名稱 | 餐廳週末版 |
| **行業** | 選擇行業分類 | 餐廳 |
| **語音** | AI 語音選擇 | Jamie（女聲粵語）|
| **問候語** | AI 接通後第一句話 | 您好，我係XXX嘅Jamie... |
| **AI 腳本** | AI 對話指示 | 詳見下方說明 |
| **啟用範本 WhatsApp 確認** | 此範本的活動完成確認後，自動發送 WhatsApp 訊息給客人 | 開啟/關閉 |

**問候語及腳本的變數說明：**

您可以在問候語和腳本中使用以下變數，系統會在致電時自動替換為實際資料：

| 變數 | 說明 |
|------|------|
| `{{name}}` | 客人姓名 |
| `{{date}}` | 預訂日期 |
| `{{time}}` | 預訂時間 |
| `{{party_size}}` | 人數/備註 |
| `{{business}}` | 您的商戶名稱 |

**範例問候語（餐廳訂座確認）：**
```
您好，我係{{business}}嘅Jamie。我哋打嚟確認您
{{date}}{{time}}，{{party_size}}位嘅訂座。
請問您係咪如期到來？
```

**步驟 2.1.3 — 儲存範本**

填寫完畢後，點擊「**儲存範本**」按鈕。範本將出現在列表中，標示為「自訂」。

```
[截圖 2.1.3: 範本已儲存，列表顯示新建立的範本
帶有紫色「自訂」標籤]
```

### 2.2 建立新活動

**步驟 2.2.1 — 開始建立活動**

在主畫面點擊右上角「**+ 新增**」按鈕，或點擊頂部行業範本區的「**新增活動**」。

```
[截圖 2.2.1: 新增活動頁面，顯示四個步驟區域：
1. 範本、2. 活動、3. 預訂、4. 何時致電？]
```

---

**第一步：選擇範本**

```
[截圖 2.2.2: 第一步範本選擇，顯示下拉選單和範本腳本預覽]
```

1. 點擊下拉選單，選擇您要使用的範本（例如「🍽️ 餐廳」）
2. 選擇後，下方會顯示該範本的腳本預覽
3. 若需要新增或修改範本，點擊右側「**管理範本**」連結

> **提示：** 系統已預設 6 個行業範本（餐廳、美容院、保險、旅行社、醫療診所、地產），開箱即用。

---

**第二步：活動設定**

```
[截圖 2.2.3: 第二步活動設定，顯示活動名稱、
預訂日期和預訂時間欄位]
```

| 欄位 | 說明 |
|------|------|
| **活動名稱** | 系統自動填入（格式：DDMMYYYYHHMMSS），可自行修改 |
| **預訂日期** | 所有聯絡人的預設預訂日期（例如：2026-05-31）|
| **預訂時間** | 所有聯絡人的預設預訂時間（例如：07:00 PM）|

> **注意：** 預訂日期和時間會對應腳本中的 `{{date}}` 和 `{{time}}`。若個別聯絡人有不同時間，可在第三步的「預訂時間」欄位單獨設定。

---

**第三步：新增預訂記錄**

```
[截圖 2.2.4: 第三步預訂記錄，顯示「Booking 1」輸入表單，
包含客人姓名、電話（附區碼選擇器）、
預訂時間、預訂日期和備註欄位]
```

每筆預訂記錄包含以下欄位：

| 欄位 | 說明 | 範例 |
|------|------|------|
| **客人姓名** | 致電時使用 `{{name}}` 替換 | 陳先生 |
| **電話** | 附區碼選擇器，預設 🇭🇰 +852 | 51873117 |
| **預訂時間** (選填) | 覆蓋活動預設時間 | 7pm |
| **預訂日期** (選填) | 覆蓋活動預設日期 | 30-May |
| **備註** (選填) | 對應 `{{party_size}}` | 4位，靠窗 |

**新增多筆預訂：**

點擊「**+ 新增預訂**」按鈕，可新增更多聯絡人。

```
[截圖 2.2.5: 顯示多筆預訂記錄，以及「+ 新增預訂」按鈕]
```

**批量匯入聯絡人（CSV）：**

若聯絡人較多，可使用 CSV 檔案批量匯入：

2. 用 Excel 或 Numbers 打開，填寫資料後儲存
3. 點擊「**匯入 CSV**」上傳檔案

```
[截圖 2.2.6: CSV 範本格式，顯示五欄標題：
Name, Telephone, Schedule, Date, Remark]

王美華,98765432,,,靠窗
```

**批量匯入聯絡人（圖片）：**

您也可以直接上傳手寫或印刷的聯絡人名單圖片：

1. 點擊「**匯入圖片**」
2. 選擇聯絡人名單的相片（支援手寫名單）
3. AI 自動識別並提取姓名、電話及備註

```
[截圖 2.2.7: 圖片匯入功能，顯示上傳按鈕及
AI 正在提取聯絡人的載入狀態]
```

---

**第四步：設定致電時間**

```
[截圖 2.2.8: 第四步致電時間選擇，顯示兩個選項卡：
「立即開始（確認後立即開始致電）」

```

**選項一 — 立即開始：**
點擊「**立即開始**」卡片（綠色邊框），確認後 AI 即時開始致電所有聯絡人。

**選項二 — 排程致電：**
點擊「**排程致電**」卡片，在下方出現的日期時間選擇器中設定致電時間。

---

**確認並啟動活動：**

```
[截圖 2.2.9: 頁面底部的活動摘要欄，顯示：
✅ 準備啟動 / ⚠️ 新增預訂以啟動
聯絡人數量・日期時間・範本名稱

```

頁面底部會顯示活動摘要：
- ✅ 綠色代表已填寫必要資料，可以啟動

確認無誤後，點擊「**確認並啟動活動**」按鈕。

**步驟 2.3.1 — 活動列表**

回到主畫面，活動列表顯示所有活動的狀態：

[截圖 2.3.1: 主畫面活動列表，顯示「進行中」標籤頁，

**步驟 2.3.3 — 活動操作**

在活動詳情頁右上角點擊「**···**」可進行以下操作：

```
[截圖 2.3.7: 操作選單展開，顯示所有操作選項]
```

| 操作 | 說明 |
|------|------|

| **暫停** | 暫停正在進行的活動 |
| **繼續** | 繼續已暫停的活動 |

### 2.4 複製活動（重複使用）

若您需要對相同聯絡人群體定期致電（例如每週確認訂座），可使用「複製活動」功能：

系統會建立一個包含相同腳本和語音設定的新草稿，並自動跳轉至聯絡人匯入頁面。

```
[截圖 2.4.1: 複製後的新活動頁面，顯示
活動名稱帶有「(copy)」後綴，處於草稿狀態]
```

**步驟 2.4.2 — 新增新一批聯絡人**

匯入新的聯絡人 CSV 或逐一輸入，然後啟動活動。

## 3. 入線熱線設定

入線熱線讓 AI 自動接聽客戶來電，回答問題、接受訂位，並在非辦公時間留言。

### 3.1 建立入線熱線

**步驟 3.1.1 — 進入入線管理**

在左側導航點擊「**入線**」。

```
[截圖 3.1.1: 入線頁面，顯示現有熱線列表
及右上角「+ 新增熱線」按鈕]
```

**步驟 3.1.2 — 新增熱線**

點擊「**+ 新增熱線**」按鈕。

```
[截圖 3.1.2: 建立入線熱線表單，顯示各設定欄位]
```

填寫以下資料：

**基本設定：**

| 欄位 | 說明 | 範例 |
|------|------|------|
| **行業範本** | 選擇預設行業（自動填入系統提示）| 🍽️ 餐廳 |
| **熱線名稱** * | 識別用名稱 | 主要訂座熱線 |
| **Twilio 號碼** * | 附區碼選擇器 + 電話號碼 | 🇭🇰 +852 / 26715377 |
| **語音** | AI 語音選擇 | Jamie（女聲粵語）|

**AI 設定：**

| 欄位 | 說明 |
|------|------|
| **系統提示** | AI 的行為指示，說明如何回應客人 |
| **營業時間** | 設定辦公時間（預設週一至週六）|
| **非營業時間 AI 指示** | 辦公時間外的 AI 回應方式 |

**系統提示範例（餐廳）：**
```
你係{{business}}嘅友善主持人。協助來電者處理訂座、
查詢營業時間、菜單問題及特別要求。保持簡潔、
熱情及專業。如有需要請轉交員工跟進。
```

**步驟 3.1.3 — 儲存熱線**

點擊「**建立熱線**」按鈕。

```
[截圖 3.1.3: 建立成功後跳轉至熱線詳情頁，
顯示「啟用中」狀態標籤及四個標籤頁]
```

**步驟 3.1.4 — 設定 Twilio 號碼 Webhook**

在 Twilio 後台，將您電話號碼的語音 Webhook 設定為：
```
https://business-console.zeabur.app/api/twiml/inbound
```

> **重要：** 完成此步驟後，AI 才能接聽來電。

### 3.2 設定知識庫

知識庫讓 AI 能正確回答關於您業務的問題（營業時間、服務項目等）。

**步驟 3.2.1 — 進入知識庫標籤**

在熱線詳情頁點擊「**知識庫**」標籤。

```
[截圖 3.2.1: 知識庫標籤頁，顯示系統自動建立的預設文章
（餐廳介紹、營業時間、訂座政策、常見問題）]
```

> **提示：** 系統會根據您選擇的行業範本，自動建立 4 篇預設知識庫文章。

**步驟 3.2.2 — 編輯文章內容**

預設文章包含通用範本，您需要將 `{{business}}` 等佔位符替換為您的實際資料。

點擊任一文章卡片可進行編輯（目前版本需透過刪除後重新新增）。

**步驟 3.2.3 — 新增文章**

點擊「**+ 新增文章**」按鈕：

```
[截圖 3.2.3: 新增文章表單，顯示標題和內容欄位]
```

| 欄位 | 說明 | 範例 |
|------|------|------|
| **文章標題** | 主題標題 | 我們的菜單 |
| **文章內容** | 詳細資訊 | 我們提供廣東料理... |

> **最佳實踐：** 建議建立以下文章：
> - 商戶介紹（地址、特色）
> - 實際營業時間
> - 服務項目及收費
> - 常見問題解答

### 4.5 Twilio 訊息範本設定

WhatsApp 確認訊息透過 Twilio Content Template 發送，需在 Twilio 後台建立範本。

**步驟 4.5.1 — 在 Twilio 建立 Content Template**

1. 登入 [Twilio Console](https://console.twilio.com)
2. 前往 **Messaging → Content Template Builder**
3. 點擊「**Create new template**」
4. 選擇「**Custom**」類型
5. 填寫範本內容，使用以下**具名變數**（必須與以下名稱完全一致）：

```
📋 {{restaurant}}訂位通知

親愛的 {{customer}}，

您的訂位已{{status}}：

📅 日期：{{date}}
🕡 時間：{{time}}
👥 人數：{{people}} 位

期待您的光臨，如有任何查詢，歡迎隨時聯絡我們。謝謝！
```

> **重要：** 變數名稱必須使用 `{{restaurant}}`、`{{customer}}`、`{{status}}`、`{{date}}`、`{{time}}`、`{{people}}`，系統才能正確填入資料。

6. 提交範本待 WhatsApp 審核（通常數小時內完成）
7. 審核通過後，複製範本的 **Content SID**（格式：`HXxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`）

**步驟 4.5.2 — 設定範本 SID（進階）**

若您有多個商戶使用不同的 Twilio 範本，可在資料庫設定每個商戶專屬的 SID：

```sql
INSERT INTO app_settings (key, value)
VALUES ('wa_template_sid', 'HX你的範本SID')
ON CONFLICT (key) DO UPDATE SET value = EXCLUDED.value;
```

若未設定，系統將使用預設範本 SID（在 `lib/wa-confirmation.ts` 中設定）。

**步驟 4.5.3 — 確認訊息變數對應**

系統發送 WhatsApp 時的變數對應如下：

| 範本變數 | 資料來源 |
|----------|----------|
| `{{restaurant}}` | 設定 → 商戶名稱 |
| `{{customer}}` | 聯絡人姓名 |
| `{{status}}` | 固定為「已確認」 |
| `{{date}}` | 聯絡人的預訂日期（`custom_data.date`）或 AI 從通話中提取 |
| `{{time}}` | 聯絡人的預訂時間（`custom_data.time`）或 AI 從通話中提取 |
| `{{people}}` | 聯絡人的人數（`custom_data.party_size`）或「-」若未填寫 |

> **注意：** 若聯絡人資料中沒有結構化的日期/時間欄位（例如只有備註「7pm 2 Jun」），AI 會嘗試從通話記錄中自動提取日期和時間。

## 6. 常見問題

### Q1: AI 說的名字不正確怎麼辦？

確保在建立活動時，每位聯絡人的「**客人姓名**」欄位已正確填寫，且腳本中使用了 `{{name}}` 變數。

### Q2: 致電時間和地點為什麼是錯的？

檢查以下設定：
1. 活動的「**預訂日期**」和「**預訂時間**」是否正確
2. 腳本中是否使用了 `{{date}}` 和 `{{time}}` 變數
3. 如需個別聯絡人不同時間，需在預訂記錄的「**預訂時間**」欄位單獨設定

### Q3: 如何暫停正在進行的活動？

進入活動詳情頁，點擊右上角「**暫停**」按鈕。正在進行的通話會完成，但不會開始新的通話。

### Q4: WhatsApp 機器人沒有回應怎麼辦？

確認您的 WhatsApp 號碼已被管理員加入授權名單。您可在 Zeabur 後台的 PostgreSQL → `whatsapp_admins` 表格中查看授權號碼。

### Q5: 知識庫文章如何影響 AI 的回答？

AI 會根據知識庫中的文章回答問題。例如，若知識庫中有正確的營業時間，AI 就能準確告知客人。建議定期更新知識庫以確保資訊準確。

### Q6: 如何讓同一客人的訂座記錄不重複致電？

使用「**重設及重試**」功能前，請先確認哪些客人需要重新致電。您可以先「**複製活動**」，然後只輸入需要重致的聯絡人。

### Q7: 客人通話確認後，為什麼沒有收到 WhatsApp 確認訊息？

請逐一檢查以下設定：

1. **設定頁面**：「外撥通話確認」開關是否已開啟（紫色）
2. **活動範本**：「啟用範本 WhatsApp 確認」是否已開啟
3. **聯絡人電話**：電話號碼必須包含國家代碼（例如 +85251873117）
4. **Twilio 範本**：Content Template 是否已審核通過，SID 是否正確
5. **通話結果**：通話報告的「AI outcome」是否顯示為「booking_confirmed」

> **提示：** 在通話報告頁查看「✅ WhatsApp 確認已發送」標籤，確認訊息是否成功發送。

### Q8: WhatsApp 訊息顯示的日期/時間/人數不正確怎麼辦？

系統優先使用聯絡人資料中的結構化欄位（`date`、`time`、`party_size`）。如果您的聯絡人只有備註（例如「7pm 2 Jun」），系統會嘗試從通話記錄中自動提取。

建議在建立活動時，盡量使用「**預訂日期**」和「**預訂時間**」欄位填寫格式化資料（例如 `2026-06-02` 和 `19:00`），而非在備註中輸入自由文字。
